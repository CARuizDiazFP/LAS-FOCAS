# PR Diario - 2026-04-17

## Resumen de Cambios

Se normalizó el ecosistema agéntico de `.github/` con descripciones orientadas a discovery, nuevas referencias reutilizables y el workflow `repo-updater` para trazabilidad Git. En paralelo se ajustó la UI de Infra para manejar avisos por baneo individual, mostrar el total de cámaras restringidas y exponer ese conteo desde el endpoint web.

## Contexto y Alcance

- **Módulos afectados**: `.github/agents/`, `.github/prompts/`, `.github/skills/`, `web/`, `docs/`, `AGENTS.md`
- **Objetivo**: ordenar el ecosistema agéntico, formalizar un workflow autónomo de sincronización y mejorar la operación del Protocolo de Protección en Infra
- **Supuestos**: `repo-updater` usará la CLI real de `git` y los artefactos temporales locales (`temp_*`, `test.db`) no forman parte del cambio funcional a subir

## Cambios Realizados

- `.github/agents/*.agent.md`: normalización de `description`, `argument-hint` y `tools` para mejorar descubrimiento y consistencia de agentes por dominio
- `.github/agents/skill-generator.agent.md`: nuevo meta-agente para crear y estandarizar skills, prompts y agentes
- `.github/prompts/*.prompt.md`: migración de prompts a un formato homogéneo con `description`, `argument-hint`, `agent` y objetivos explícitos
- `.github/prompts/crear-skill.prompt.md`: nuevo prompt para generar skills o tríadas completas de customizations
- `.github/prompts/repo-updater.prompt.md`: nuevo prompt para auditar trazabilidad, construir commit técnico y hacer push a `main`
- `.github/skills/*`: simplificación de skills existentes, incorporación de referencias reutilizables y alta de `repo-updater` y `skill-generator`
- `.github/skills/repo-update/SKILL.md`: conversión a alias legacy para evitar duplicación de workflows
- `web/templates/panel.html`, `web/static/panel.js`, `web/static/styles.css`: eliminación del botón global de aviso, agregado de botón individual por baneo y nuevo indicador visual de cámaras restringidas
- `web/web_app/main.py`: endpoint `/api/infra/ban/active` ahora informa `camaras_count` por incidente
- `docs/infra.md`: nueva documentación específica del módulo Infra y del flujo actualizado de baneos
- `AGENTS.md`, `docs/Mate_y_Ruta.md`, `docs/decisiones.md`: documentación alineada con el estado actual del ecosistema agéntico

## Comandos Ejecutados

- `git status --short --branch`
  - Resultado: rama local `main` con cambios sin stagear contra `origin/main`
- `git diff --stat`
  - Resultado: alcance principal en `.github/`, documentación y módulo web de Infra; sin cambios staged previos
- `git diff --name-status`
  - Resultado: identificación de archivos modificados y detección de artefactos temporales no deseados (`temp_alembic.ini`, `temp_env.py`, `test.db`)
- `read_file`, `grep_search`, `get_changed_files`
  - Resultado: contraste del diff con la documentación y ajuste del alcance real a subir

## Impacto y Riesgos

- **Impacto operativo**: mejora el discovery de customizations, reduce ambigüedad del ecosistema agéntico y vuelve más operable el módulo Infra cuando existen múltiples baneos activos
- **Riesgos conocidos**: persiste convivencia temporal del alias `repo-update`; además, los artefactos temporales locales deben quedar fuera del commit para no contaminar el repo
- **Seguridad y datos**: sin exposición nueva de secretos; el flujo de repo-updater refuerza la exclusión de credenciales y de comandos destructivos

## Compatibilidad y Migraciones

- **Breaking changes**: no aplica; `repo-update` se conserva como alias de compatibilidad y la UI de Infra mantiene el flujo funcional con mejor granularidad de avisos
