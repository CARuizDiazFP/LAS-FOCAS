# PR Diario - 2026-04-28

## Resumen de Cambios

Dos mejoras independientes sobre el subsistema de gestión de cámaras:

1. **Parser multi-bot** (`camara_search.py`, `listener.py`): cuando un técnico escribe "Botella 1 y 2" (o variantes como "Bot 1 y 2", "Botellas 2 y 3") en un mismo mensaje de Slack, el listener detecta el patrón y realiza dos búsquedas independientes en DB — una por cada cámara —, respondiendo en el mismo hilo con ambos estados concatenados. Elimina el problema de auto-registrar un nombre combinado basura como `PENDIENTE_REVISION`.

2. **Eliminar cámaras pendientes desde el panel admin** (`main.py`, `admin.ts`, `AdminBaneos.vue`): nuevo endpoint `DELETE /api/admin/infra/camaras/pendientes/{id}` y botón "🗑️ Eliminar" con confirmación inline en la tabla de pendientes. Permite limpiar registros erróneos sin necesidad de intervención directa en DB.

## Contexto y Alcance

- **Módulos afectados**: `modules/slack_baneo_notifier/`, `web/web_app/`, `web/frontend/src/admin/`
- **Causa raíz multi-bot**: input "Bartolomé Mitre 301. Botella 1 y 2. CF" llegaba intacto a `buscar_camara` y, al no encontrar nada, se auto-registraba como un único nombre combinado basura.
- **Causa raíz eliminar**: no existía forma de borrar registros `PENDIENTE_REVISION` desde el panel; había que hacerlo por DB directamente.

## Cambios Realizados

### modules/slack_baneo_notifier/camara_search.py *(modificado)*

- **Nuevo regex** `_RE_MULTI_BOT`:
  ```python
  re.compile(r"(?i)\bbot(?:ella)?s?\s+(\d+)\s+(?:y|&)\s+(\d+)\b")
  ```
- **Nueva función exportada** `detectar_multi_bot(nombre_raw: str) -> list[str] | None`:
  - Detecta el patrón, extrae los dos números `N` y `M`.
  - Limpia el patrón de la cadena base.
  - Para N=1: retorna la base sin prefijo (Botella 1 = cámara principal en DB).
  - Para N≥2: retorna `"Bot N " + base`.
  - Retorna `None` si el patrón no está presente.
  - Ejemplo: `"Bartolomé Mitre 301. Botella 1 y 2. CF"` → `["Bartolomé Mitre 301 CF", "Bot 2 Bartolomé Mitre 301 CF"]`.

### modules/slack_baneo_notifier/listener.py *(modificado)*

- Import actualizado para incluir `detectar_multi_bot`.
- **Nuevo método privado** `_construir_respuesta_camara(nombre_buscado, session)`:
  - Llama a `buscar_camara()`.
  - Si no se encuentra: auto-registra como `PENDIENTE_REVISION` y retorna mensaje de revisión.
  - Si se encuentra con incidentes activos: retorna alerta de baneo.
  - Si se encuentra sin incidentes: retorna mensaje de habilitación.
- **`_handle_message` refactorizado**:
  - Después de extraer `nombre_raw`, llama a `detectar_multi_bot(nombre_raw)`.
  - Si multi-bot detectado: itera sobre los nombres generados llamando a `_construir_respuesta_camara` por cada uno.
  - Si no: flujo normal con `[nombre_raw]`.
  - Une las respuestas con `"\n\n─────────────────────\n\n"` y envía un único `chat_postMessage`.

### web/web_app/main.py *(modificado)*

- **Nuevo endpoint** `DELETE /api/admin/infra/camaras/pendientes/{camara_id}`:
  - Requiere sesión de administrador (`_require_admin`).
  - Retorna 404 si el registro no existe.
  - Retorna 409 si el estado NO es `PENDIENTE_REVISION` (guard de seguridad).
  - Borra físicamente el registro con `session.delete()` + `session.commit()`.
  - Log: `action=eliminar_pendiente camara_id=... nombre='...'`.
  - Respuesta: `{"ok": True, "id": camara_id}`.

### web/frontend/src/admin/api/admin.ts *(modificado)*

- **Nueva función exportada** `eliminarCamaraPendiente(id: number): Promise<void>`:
  - Llama a `DELETE /api/admin/infra/camaras/pendientes/{id}` con `credentials: 'include'`.
  - Lanza `Error` con el mensaje del servidor si la respuesta no es OK.

### web/frontend/src/admin/views/AdminBaneos.vue *(modificado)*
